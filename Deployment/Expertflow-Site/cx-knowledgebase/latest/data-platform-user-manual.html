Data Platform User Manual

In case of private FQDNs, you need to register the IP and FQDN in the hosts file located in drivers/etc directory of the local machine/system

Once the pod is up (after 1 minute), access the Data Platform UI using the allocated FQDN and login using admin.

Initially, pipelines will be turned off (slider) as shown in Figure 01. Turn on the pipelines as per the following order:

  1. Forms Forms Data Pipeline

  2. Teams Teams Data Pipeline

Quality Management and Campaigns are dependent on the Forms and Teams pipeline.

  1. For Quality Management

    1. Quality_Management QM Data Pipeline

    2. Keycloak_users Keycloak Users Pipeline

    3. Cisco_Agent_Team_Sync Cisco Agent & Team Sync Pipeline

  2. For Campaigns Campaigns Data Pipeline

    1. Campaigns

    2. Activities

    3. gold_15_minute_queries

  3. For bulk upload Customer Bulk Upload ETL

    1. fetch_upload_records

    2. process_upload_records

  4. For Graphana SLA Grafana SLA, AWHT Pipeline

    1. graphana_queries

All pipelines are set at scheduled intervals, and will start running once the pipelines are turned on (turn the slider right)
